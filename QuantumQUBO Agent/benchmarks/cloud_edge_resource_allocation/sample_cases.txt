CLOUD/EDGE RESOURCE ALLOCATION TEST CASES

Source inspiration:
Carlo Mastroianni, Francesco Plastina, Jacopo Settino, and Andrea Vinci,
"Variational Quantum Algorithms for the Allocation of Resources in a Cloud/Edge
Architecture",
IEEE Transactions on Quantum Engineering 5, pp. 1-17, 2024.
DOI: https://doi.org/10.1109/TQE.2024.3398410

Benchmark simplification:
These cases use small task-to-node assignment costs and integer capacities.


========================================================================
TEST CASE 1: Three tasks, one edge and one cloud node

Nodes:
- Edge_E0: capacity 2
- Cloud_C0: capacity 3

Tasks and assignment costs:
- T0: Edge_E0=1, Cloud_C0=3
- T1: Edge_E0=2, Cloud_C0=2
- T2: Edge_E0=4, Cloud_C0=1

Ground truth:
- Minimum allocation cost: 4
- Number of optimal assignments: 2
- Optimal assignments:
  - T0 -> Edge_E0, T1 -> Edge_E0, T2 -> Cloud_C0
  - T0 -> Edge_E0, T1 -> Cloud_C0, T2 -> Cloud_C0


========================================================================
TEST CASE 2: Two edge nodes and one cloud fallback

Nodes:
- Edge_E0: capacity 1
- Edge_E1: capacity 1
- Cloud_C0: capacity 3

Tasks and assignment costs:
- T0: Edge_E0=1, Edge_E1=4, Cloud_C0=3
- T1: Edge_E0=2, Edge_E1=1, Cloud_C0=3
- T2: Edge_E0=4, Edge_E1=2, Cloud_C0=1

Ground truth:
- Minimum allocation cost: 3
- Number of optimal assignments: 1
- Assignments:
  - T0 -> Edge_E0
  - T1 -> Edge_E1
  - T2 -> Cloud_C0
