Score-Based Bayesian Network Structure Learning with Candidate Parent Sets

We are going to learn the structure of a Bayesian network. The network is a
directed graph on ordered nodes. For every node, we must choose exactly one
candidate parent set. The chosen parent sets define the directed edges of the
network.

The nodes have a fixed order, such as 0 < 1 < 2. A node can only have parents
that come before it in this order. For node i, every parent set must be a subset
of {0, 1, ..., i-1}. Because parents only point from earlier nodes to later
nodes, the final graph has no directed cycles.

Each node has its own list of candidate parent sets. A candidate parent set also
has a score. Different nodes may have different numbers of candidate parent
sets.

Only the candidate parent sets listed for a node can be selected for that node.
Do not use any parent set that is not listed in the input.

If a selected parent set has r parents, it contributes r directed edges into
that node. The final network must have exactly K directed edges in total.

The total score is the sum of the scores of the selected parent sets. The task
is a maximization problem.

Input:
- The fixed node order.
- K, the required total number of directed edges in the final network.
- For each node i:
  - the candidate parent sets for node i, and
  - the score of each candidate parent set.

Goal:
Choose exactly one candidate parent set for every node so that the final network
has exactly K directed edges and the total score is as large as possible.
