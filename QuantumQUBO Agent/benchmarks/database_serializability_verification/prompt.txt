Database Transaction Serializability Verification Problem

Given a set of database transactions and observed read/write dependency
constraints, decide whether the observed execution can be serialized. In this
benchmark, candidate transaction orders are provided with violation costs.

Task:
Choose one candidate serial order.

Objective:
Minimize the number of dependency violations.

Goal:
Return the selected transaction order and minimum violation count. A violation
count of 0 means the execution is serializable under the supplied constraints.

QUBO convention:
Use binary variables to represent ordering decisions or candidate order
selection. Penalties encode dependency constraints such as transaction A must
precede transaction B.
