TRAVELLING SALESMAN TEST CASES

Source inspiration:
Andrew Lucas,
"Ising formulations of many NP problems",
Frontiers in Physics 2, Article 5, 2014.
DOI: https://doi.org/10.3389/fphy.2014.00005

Benchmark simplification:
These instances are kept at N=4 cities so the natural city-position encoding
(x_{c,p}) yields 16 binary variables, allowing exact 2^n verification.


========================================================================
Test case: 1  (N=4)
Source: GeeksforGeeks — Travelling Salesman Problem using Dynamic Programming
        https://www.geeksforgeeks.org/dsa/travelling-salesman-problem-using-dynamic-programming/

Cities: 4  (labelled 0 to 3)

Distance matrix D[i][j] (symmetric, D[i][i] = 0):
         0    1    2    3
  0      0   10   15   20
  1     10    0   35   25
  2     15   35    0   30
  3     20   25   30    0

This is a minimisation problem (minimise total tour distance).
Minimum tour length = 80.
QUBO minimisation optimal value = 80.

There is exactly one optimal undirected tour:
  0 - 1 - 3 - 2 - (back to 0)  total distance = 10 + 25 + 30 + 15 = 80
  (GeeksforGeeks states this as 0 -> 2 -> 3 -> 1 -> 0, which is the same tour traversed in reverse.)

All other tours are strictly longer:
  0 - 1 - 2 - 3: 10 + 35 + 30 + 20 = 95
  0 - 2 - 1 - 3: 15 + 35 + 25 + 20 = 95

Test case: 2  (N=4) — symmetric ring with one shortcut

Cities: 4  (labelled 0 to 3)

Distance matrix D[i][j] (symmetric, D[i][i] = 0):
         0    1    2    3
  0      0    1    5    1
  1      1    0    1    5
  2      5    1    0    1
  3      1    5    1    0

This is a minimisation problem (minimise total tour distance).
Minimum tour length = 4.
QUBO minimisation optimal value = 4.

The optimal tour follows the unit-cost ring:
  0 - 1 - 2 - 3 - (back to 0)  total distance = 1 + 1 + 1 + 1 = 4
