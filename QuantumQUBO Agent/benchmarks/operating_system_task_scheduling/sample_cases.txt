OPERATING-SYSTEM TASK SCHEDULING TEST CASES

Source inspiration:
Tian Huang, Yongxin Zhu, Rick Siow Mong Goh, and Tao Luo,
"When quantum annealing meets multitasking: Potentials, challenges and
opportunities",
Array 17, 100282, 2023.
DOI: https://doi.org/10.1016/j.array.2023.100282

Jargalsaikhan Artag, Koki Awaya, Takumi Kanezashi, Daisuke Tsukayama, Moe
Shimada, and Jun-ichi Shirakashi,
"Multi-tasking through quantum annealing",
EPJ Quantum Technology, 2026.
DOI: https://doi.org/10.1140/epjqt/s40507-026-00504-z

Benchmark simplification:
The source papers study multitasking, resource sharing, task scheduling, and
preemption-like execution for quantum annealing systems. These benchmark cases
use classical operating-system style time slices and context-switch costs as a
small QUBO scheduling analogue.


========================================================================
TEST CASE 1: Three tasks, two time slices

Time slices:
- t0
- t1

Tasks:
- A0:
  - class: interactive
  - priority reward: 8
- A1:
  - class: batch
  - priority reward: 5
- A2:
  - class: interactive
  - priority reward: 6

Context-switch penalties between adjacent slices:
- interactive -> batch: 2
- batch -> interactive: 2
- interactive -> interactive: 0
- batch -> batch: 0

Ground truth:
- Maximum net score: 14
- QUBO minimization objective value: -14
- Number of optimal schedules: 2
- Optimal schedules:
  - t0 -> A0, t1 -> A2
  - t0 -> A2, t1 -> A0
- Score calculation:
  - priority reward: 8 + 6 = 14
  - context-switch penalty: 0
  - net score: 14


========================================================================
TEST CASE 2: Priority task with switch penalty

Time slices:
- t0
- t1
- t2

Tasks:
- A0:
  - class: interactive
  - priority reward: 9
- A1:
  - class: batch
  - priority reward: 7
- A2:
  - class: batch
  - priority reward: 6
- A3:
  - class: realtime
  - priority reward: 8

Context-switch penalties between adjacent slices:
- same class: 0
- interactive with batch: 2
- interactive with realtime: 1
- batch with realtime: 3

Ground truth:
- Maximum net score: 21
- QUBO minimization objective value: -21
- Number of optimal schedules: 2
- Optimal schedules:
  - t0 -> A1, t1 -> A0, t2 -> A3
  - t0 -> A3, t1 -> A0, t2 -> A1
- Score calculation:
  - priority reward: 9 + 8 + 7 = 24
  - context-switch penalty: 2 + 1 = 3
  - net score: 21


========================================================================
TEST CASE 3: Batch-friendly schedule beats higher switching

Time slices:
- t0
- t1
- t2

Tasks:
- A0:
  - class: batch
  - priority reward: 7
- A1:
  - class: batch
  - priority reward: 6
- A2:
  - class: interactive
  - priority reward: 8
- A3:
  - class: realtime
  - priority reward: 6

Context-switch penalties between adjacent slices:
- same class: 0
- batch with interactive: 3
- batch with realtime: 2
- interactive with realtime: 4

Ground truth:
- Maximum net score: 18
- QUBO minimization objective value: -18
- Number of optimal schedules: 4
- Optimal schedules:
  - t0 -> A0, t1 -> A1, t2 -> A2
  - t0 -> A1, t1 -> A0, t2 -> A2
  - t0 -> A2, t1 -> A0, t2 -> A1
  - t0 -> A2, t1 -> A1, t2 -> A0
- Score calculation:
  - priority reward: 7 + 6 + 8 = 21
  - context-switch penalty: 0 + 3 = 3 for A0,A1,A2
  - net score: 18
