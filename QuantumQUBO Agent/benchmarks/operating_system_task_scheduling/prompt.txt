Operating-System Task Scheduling Problem

Given tasks, time slices, and processor/resource slots, assign tasks to time
slices. Each task has a priority reward, and switching between task classes can
incur a context-switch cost.

Task:
Choose a feasible schedule over the available time slices.

Constraints:
- Each time slice can run at most one task.
- A task can be scheduled at most once in these small benchmark cases.

Objective:
Maximize scheduled priority reward minus context-switch penalties.

Goal:
Return the selected task schedule and maximum net score.

QUBO convention:
Use one binary variable per task-time assignment. One-hot and at-most-one
penalties enforce feasibility. Quadratic terms encode context-switch costs
between tasks assigned to adjacent time slices.
