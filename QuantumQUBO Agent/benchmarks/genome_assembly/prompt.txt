DNA Read Ordering Problem for Genome Assembly

In genome assembly, a DNA sequence is often reconstructed from short fragments called reads.

A read is a short piece of DNA obtained from a longer original DNA sequence. For example, if the original sequence is:

ATGCAT

then possible short reads of length 3 are:

ATG  
TGC
GCA  
CAT

These reads overlap with each other. An overlap happens when the ending part of one read matches the starting part of another read.

Example:

Read 1: ATG
Read 2: TGC

The suffix "TG" of ATG matches the prefix "TG" of TGC.

So the overlap from ATG to TGC is 2.

Another example:

Read 1: GCA
Read 2: CAT

The suffix "CA" of GCA matches the prefix "CA" of CAT.

So the overlap from GCA to CAT is 2.

The pairwise overlap matrix O stores these overlap values. Each entry O[i][j] represents the length of the longest suffix of read i that matches a prefix of read j.

For example, if:

r0 = ATG
r1 = TGC
r2 = GCA
r3 = CAT

then O[0][1] = 2 because ATG overlaps with TGC by "TG".

The problem is to arrange all reads in a single linear order so that the total overlap between consecutive reads is as large as possible.

For example, the ordering:

ATG -> TGC -> GCA -> CAT

has overlaps:

ATG -> TGC = 2
TGC -> GCA = 2
GCA -> CAT = 2

Total overlap = 2 + 2 + 2 = 6

A high-overlap ordering is useful because it helps reconstruct the original DNA sequence by stitching the reads together end-to-end. The better the consecutive overlaps, the more likely the ordering represents the original genome sequence.

Given:
- A set of N DNA short reads
- A pairwise overlap matrix O
- O[i][j] = length of the longest suffix of read i matching a prefix of read j

Task:
Find a linear ordering of all reads that maximizes the total overlap between every pair of consecutive reads in the ordering using 
quadratic unconstrained binary optimization (QUBO).

Goal:
Maximize the sum of overlap scores between adjacent reads in the final ordering.
